# Decision 034: Use One Shared Theme and Semantic UI Icons

- **Date:** 2026-07-14
- **Status:** Accepted

## Context

The combat HUD and character menu had useful layouts but duplicated most panel styling and identified actions only through text. Upcoming title, Sanctuary, dialogue, and shop screens need a coherent visual baseline without coupling gameplay rules to particular image filenames.

## Alternatives Considered

- Continue styling every scene locally.
- Build one large UI atlas and address icons by coordinates in gameplay scripts.
- Use a shared Godot `Theme` plus separately named semantic icon textures.

## Decision

Battle of Gods uses `battle_of_gods_theme.tres` as its reusable base for common controls. Small runtime icons have stable semantic IDs and individual files at their native 16x16 or 24x24 size. Scenes or presentation metadata provide textures to UI; gameplay rules never branch on texture paths.

The initial icon kit is reproducibly generated by `build_ui_icon_kit.gd` with the approved palette and binary alpha. Contextual interaction events may include a presentation icon, allowing portals and future NPCs to share one HUD prompt.

## Consequences

- New screens inherit a consistent base and override only genuinely semantic states.
- Individual icons can be replaced without repacking an atlas or changing gameplay code.
- A few existing local styles remain temporarily for health, cooldown, ready, and locked meaning; these should converge gradually rather than through a risky unrelated rewrite.
- Generated icons remain intentionally simple and may be redrawn later while retaining their canonical paths and IDs.
